Get your first HelixDB instance running and execute your first query in under 5 minutes.

## Prerequisites

<Warning>
Before you begin, ensure you have:
- **Docker** installed and running (for local deployment)
- **curl** or a package manager (npm, brew, etc.)
- Basic knowledge of databases and query languages
</Warning>

## Install the CLI

<Steps>
  <Step title="Install the Helix CLI">
    The fastest way to get started is using the install script:

    ```bash
    curl -sSL "https://install.helix-db.com" | bash
    ```

    Or install manually:

    <CodeGroup>
    ```bash npm
    npm install -g helix-cli
    ```

    ```bash brew
    brew install helixdb/tap/helix
    ```

    ```bash cargo
    cargo install helix-cli
    ```
    </CodeGroup>

    Verify the installation:

    ```bash
    helix --version
    ```
  </Step>

  <Step title="Initialize a new project">
    Create a new directory for your project and initialize it:

    ```bash
    mkdir my-helix-project
    cd my-helix-project
    helix init
    ```

    This creates:
    - `helix.toml` - Project configuration
    - `db/schema.hx` - Schema definitions
    - `db/queries.hx` - Query definitions
    - `.gitignore` - Git ignore rules
  </Step>

  <Step title="Define your schema">
    Open `db/schema.hx` and define your first data model:

    ```hql db/schema.hx
    N::User {
        INDEX email: String,
        name: String,
        age: U32
    }

    E::Follows {
        From: User,
        To: User,
        Properties: {
            since: Date
        }
    }

    V::Document {
        INDEX title: String,
        content: String
    }
    ```

    <Info>
    - `N::` defines nodes in the graph
    - `E::` defines edges (relationships)
    - `V::` defines vectors for semantic search
    - `INDEX` creates fast lookup indices
    </Info>
  </Step>

  <Step title="Write your queries">
    Open `db/queries.hx` and add queries to interact with your data:

    ```hql db/queries.hx
    QUERY CreateUser(email: String, name: String, age: U32) =>
        user <- AddN<User>({email: email, name: name, age: age})
        RETURN user

    QUERY GetUserByEmail(email: String) =>
        user <- N<User>({email: email})
        RETURN user

    QUERY CreateFollowRelationship(follower_id: ID, following_id: ID) =>
        follower <- N<User>(follower_id)
        following <- N<User>(following_id)
        edge <- AddE<Follows>::From(follower)::To(following)::VALUES({since: NOW})
        RETURN edge

    QUERY GetFollowers(user_id: ID) =>
        user <- N<User>(user_id)
        followers <- user::InE<Follows>::FromN
        RETURN followers

    QUERY SearchDocuments(query: String) =>
        results <- SearchV<Document>(Embed(query), 10)
        RETURN results
    ```
  </Step>

  <Step title="Build and start your instance">
    Build and start your local HelixDB instance:

    ```bash
    helix build
    helix push dev
    ```

    Your database is now running on `http://localhost:6969`!

    <Tip>
    The first build may take a few minutes as Docker pulls the necessary images.
    </Tip>
  </Step>

  <Step title="Call your queries">
    Now you can interact with your database using the SDKs or curl:

    <Tabs>
      <Tab title="TypeScript">
        Install the SDK:

        ```bash
        npm install helix-ts
        ```

        Use it in your code:

        ```typescript
        import HelixDB from "helix-ts";

        const client = new HelixDB();

        // Create a user
        const user = await client.query("CreateUser", {
          email: "alice@example.com",
          name: "Alice",
          age: 30
        });

        console.log("Created user:", user);

        // Get user by email
        const foundUser = await client.query("GetUserByEmail", {
          email: "alice@example.com"
        });

        console.log("Found user:", foundUser);
        ```
      </Tab>

      <Tab title="Python">
        Install the SDK:

        ```bash
        pip install helix-py
        ```

        Use it in your code:

        ```python
        from helix import HelixDB

        client = HelixDB()

        # Create a user
        user = client.query("CreateUser", {
            "email": "alice@example.com",
            "name": "Alice",
            "age": 30
        })

        print("Created user:", user)

        # Get user by email
        found_user = client.query("GetUserByEmail", {
            "email": "alice@example.com"
        })

        print("Found user:", found_user)
        ```
      </Tab>

      <Tab title="cURL">
        ```bash
        # Create a user
        curl -X POST http://localhost:6969/query/CreateUser \
          -H "Content-Type: application/json" \
          -d '{
            "email": "alice@example.com",
            "name": "Alice",
            "age": 30
          }'

        # Get user by email
        curl -X POST http://localhost:6969/query/GetUserByEmail \
          -H "Content-Type: application/json" \
          -d '{
            "email": "alice@example.com"
          }'
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Manage Your Instance

View your running instances:

```bash
helix status
```

Stop your instance:

```bash
helix stop dev
```

View logs:

```bash
helix logs dev
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 6969 is already in use">
    Change the port in `helix.toml`:

    ```toml
    [instances.dev]
    port = 7000
    ```

    Then rebuild:

    ```bash
    helix build
    helix push dev
    ```
  </Accordion>

  <Accordion title="Docker daemon not running">
    Ensure Docker Desktop is running:

    ```bash
    # macOS
    open -a Docker

    # Linux
    sudo systemctl start docker
    ```
  </Accordion>

  <Accordion title="Compilation errors">
    Run `helix check` to validate your HQL files:

    ```bash
    helix check
    ```

    This will show detailed error messages with line numbers.
  </Accordion>

  <Accordion title="Connection refused">
    Verify the instance is running:

    ```bash
    helix status
    ```

    If not running, start it:

    ```bash
    helix start dev
    ```
  </Accordion>
</AccordionGroup>
